## Design Notes

### Typography for Margaret
- **Headers:** 24px, bold, with emoji
- **Body text:** 16px, comfortable line spacing (1.6)
- **Numbers:** Larger, bold, colored (green for good, amber for caution)
- **Key terms:** Bold, followed by explanation in parentheses

### Colors
- **Green (#10B981):** Positive, healthy, good news
- **Amber (#F59E0B):** Caution, needs attention
- **Red (#EF4444):** Warning, urgent (rarely used)
- **Blue (#3B82F6):** Neutral information, charts

### Charts
- **Font size:** 14-16px minimum
- **Lines:** 2-3px thick
- **Tooltips:** Show on hover with plain English
- **Max data points:** 5-7 (more is overwhelming)

### Text Structure
1. **Headline answer** (one sentence)
2. **Why/Context** (2-3 sentences)
3. **Details** (bullets or short paragraphs)
4. **Action item** (one thing to do next)

### Length Guidelines
- **Total response:** 150-250 words max
- **Paragraphs:** 2-3 sentences each
- **Bullet lists:** 3-5 items max
- **If longer needed:** Break into multiple Q&A

```json
{
  "visualType": "health-check" | "performance" | "peer-comparison" | "calculator" | "scenario" | "none",
  "data": {
    // Chart-specific data structure
  },
  "markdown": "The formatted text response"

```

The frontend will:
1. Render markdown text with `react-markdown`
2. Show appropriate chart/card based on `visualType`
3. Layout text + visual side-by-side (desktop) or stacked (mobile)
